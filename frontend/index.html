Endpoint Method Description Parameters Formats
http://176.98.237.4/ring_api/api/v1/register/ POST Registers a new user along with their associated company. This endpoint is crucial for creating new accounts and linking them to specific company data. username, email, password, company (founder, phoneNumber, companyName) Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/login/ POST Authenticates an existing user by checking the provided username and password against the database. Successful authentication returns a token for accessing protected routes. username, password Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/newToken/ POST Generates a new authentication token for an already authenticated user. This is useful for maintaining or renewing user sessions. username, password Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/assistant/ GET, POST GET method retrieves all assistants related to the user's company, providing an overview. POST method allows the creation of a new assistant, expanding the user's set of tools. (GET) None, (POST) company_id, content, name Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/assistant/<slug:name>/ GET, PUT, DELETE Manages a specific assistant by name. GET retrieves details, PUT updates certain attributes, and DELETE removes the assistant from the system. (GET, DELETE) None, (PUT) content (partial updates allowed) Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/text/ GET Processes textual queries through an AI assistant. This endpoint is designed to interact with the assistant via text, obtaining AI-driven responses based on the assistant's training and data. assistant, request Content-Type: application/json
http://176.98.237.4/ring_api/api/v1/voice/<str:assistant>/ POST Accepts voice input in the form of an OGG audio file, processes it to extract text, and uses the AI model to generate a response. The response is then converted into MP3 format and returned as an audio file to the user. Binary data (audio file) Input: audio/ogg; Output: audio/mpeg
